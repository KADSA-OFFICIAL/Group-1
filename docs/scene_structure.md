# 씬 구조 가이드

이 프로젝트는 여러 명이 동시에 작업물을 합치기 쉽도록 `Main` 씬에서 파트를 조립하는 방식으로 시작합니다.

## 씬 역할

- `scenes/main/main.tscn`: 게임 실행 진입점입니다. 배경, 플레이어, UI, 게임 상태를 조립합니다.
- `scenes/background/school_floor_1.tscn`: 학교 맵, 배경 오브젝트, 벽 충돌, 교실별 상호작용 물건을 담당합니다.
- `scenes/player/player.tscn`: 주인공 이동, 카메라, 상호작용 범위를 담당합니다.
- `scenes/ui/hud.tscn`: 목표, 알림 메시지, 소지품 표시를 담당합니다.

## 협업 규칙

- 배경 담당자는 `scenes/background`와 배경 전용 리소스를 중심으로 수정합니다.
- 플레이어 담당자는 `scenes/player`와 `scripts/player`를 중심으로 수정합니다.
- UI 담당자는 `scenes/ui`와 `scripts/ui`를 중심으로 수정합니다.
- 새 퍼즐은 가능하면 별도 씬으로 만든 뒤 배경 씬에 인스턴스화합니다.
- 공통 상태가 필요하면 `scripts/game/game_state.gd`에 작은 함수로 추가하고, 직접 다른 파트의 노드를 강하게 참조하지 않습니다.

## 현재 플레이 흐름

1. 주인공은 학교 복도에서 시작합니다.
2. `E` 키로 국어 교실의 국어책, 수위실의 현관 열쇠, 창문 단서, 현관문을 조사할 수 있습니다.
3. 현관문은 `front_gate_key` 아이템이 있어야 열립니다.

이 구조 위에 교실별 퍼즐, 추격 AI, 사운드, 화면 연출을 각각 독립 씬으로 추가하면 병합 충돌을 줄일 수 있습니다.
